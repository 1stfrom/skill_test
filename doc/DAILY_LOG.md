# Daily Log

## Purpose

This file is the human-readable commit journal for the project website.

Use one record per commit whenever possible. After each commit, add one short entry that explains what changed, why it matters, and what should happen next. The helper `python3 scripts/log_commit.py` can append or update an entry from the latest Git commit.

## Entry Template

### YYYY-MM-DD - short title

- Commit: `<hash>`
- Summary: one or two sentences on what changed
- Files: main files or directories touched
- Results or impact: what improved, changed, or was learned
- Next: the next concrete step
